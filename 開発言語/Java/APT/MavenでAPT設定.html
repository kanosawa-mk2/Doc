MavenでもAPT（Annotation Processing Tool）を使用してコードを自動生成することは可能です。通常、Mavenではmaven-compiler-pluginやmaven-processor-pluginを利用してAPTを実行します。

方法1: maven-compiler-plugin の annotationProcessorPaths を使用

Maven 3.5 以降では、maven-compiler-plugin に annotationProcessorPaths を設定することで、APTを実行できます。

設定例 (pom.xml)

xml
<build>
    <plugins>
        <plugin>
            <groupId>org.apache.maven.plugins</groupId>
            <artifactId>maven-compiler-plugin</artifactId>
            <version>3.8.1</version>
            <configuration>
                <source>1.8</source>
                <target>1.8</target>
                <annotationProcessorPaths>
                    <path>
                        <groupId>com.example</groupId>
                        <artifactId>my-annotation-processor</artifactId>
                        <version>1.0.0</version>
                    </path>
                </annotationProcessorPaths>
            </configuration>
        </plugin>
    </plugins>
</build>

この方法では、annotationProcessorPaths に指定したプロセッサが自動的に実行され、コードが生成されます。

方法2: maven-processor-plugin を使用

もう一つの方法として、maven-processor-plugin を使用することもできます。特に、APTの結果を target/generated-sources/annotations 以外の場所に出力したい場合や、カスタム設定が必要な場合に便利です。

設定例 (pom.xml)

xml
<build>
    <plugins>
        <plugin>
            <groupId>org.bsc.maven</groupId>
            <artifactId>maven-processor-plugin</artifactId>
            <version>5.0</version>
            <executions>
                <execution>
                    <id>process</id>
                    <goals>
                        <goal>process</goal>
                    </goals>
                </execution>
            </executions>
            <configuration>
                <processors>
                    <processor>com.example.MyAnnotationProcessor</processor>
                </processors>
                <outputDirectory>target/generated-sources/annotations</outputDirectory>
            </configuration>
        </plugin>
    </plugins>
</build>

この方法では、手動で使用するAPTプロセッサを指定できます。

Eclipse での設定

Eclipseを使用している場合、MavenプロジェクトでAPTを有効にするためには、次の設定を確認してください。

  1. Eclipseの設定

    • Window → Preferences → Java → Compiler → Annotation Processing で「Enable annotation processing」を有効化

    • Java Compiler の Annotation Processing の「Generated source directory」を target/generated-sources/annotations に変更

  2. EclipseのMaven設定

    • Project → Properties → Maven → Annotation Processing で「Automatically configure JDT APT」にチェックを入れる

どちらの方法を選ぶべきか？

  • シンプルに設定したい場合 → maven-compiler-plugin の annotationProcessorPaths を利用

  • 柔軟な設定が必要な場合 → maven-processor-plugin を利用

どちらの方法でもMavenプロジェクトでAPTを使用できます。Eclipse上でも適切に動作するように設定を調整してください。

Maven でローカル環境にある JAR ファイルをプロジェクトに取り込む方法はいくつかあります。用途に応じて最適な方法を選んでください。

方法1: install:install-file でローカルリポジトリに登録する (推奨)

Maven のローカルリポジトリ (~/.m2/repository) に JAR を手動で追加し、通常の依存関係として扱う方法です。

手順

  1. JARファイルをローカルリポジトリにインストール

    sh
    mvn install:install-file \
    -Dfile=/path/to/your-library.jar \
    -DgroupId=com.example \
    -DartifactId=your-library \
    -Dversion=1.0.0 \
    -Dpackaging=jar

    • -Dfile → JARファイルのパス

    • -DgroupId → 一意のグループID（通常はドメイン逆順）

    • -DartifactId → アーティファクトID（ライブラリの名称）

    • -Dversion → バージョン番号

    • -Dpackaging → パッケージ形式 (jar)

  2. pom.xml に依存関係を追加

    xml
    <dependency>
    <groupId>com.example</groupId>
    <artifactId>your-library</artifactId>
    <version>1.0.0</version>
</dependency>

  3. Mavenを更新

    sh
    mvn clean install

方法2: system スコープで直接JARファイルを指定（非推奨）

JAR ファイルを lib フォルダなどに配置し、system スコープで明示的に指定する方法です。ただし、この方法は Maven の推奨する依存管理の方式ではない ため、基本的に 方法1を使うのがベター です。

手順

  1. プロジェクトの lib フォルダに JAR を配置

    bash
    /your-maven-project/
├── lib/
│   ├── your-library.jar
├── src/
├── pom.xml

  2. pom.xml に依存関係を追加

    xml
    <dependency>
    <groupId>com.example</groupId>
    <artifactId>your-library</artifactId>
    <version>1.0.0</version>
    <scope>system</scope>
    <systemPath>${project.basedir}/lib/your-library.jar</systemPath>
</dependency>

⚠ 注意

  • system スコープはリポジトリ管理されず、CI/CD 環境などで不便になるため推奨されません。

方法3: ローカルリポジトリ (.m2/repository) を別ディレクトリに設定

もし、ローカルの .m2/repository 以外のディレクトリで管理したい場合は、settings.xml を変更することで別の場所にローカルリポジトリを設定できます。

  1. ~/.m2/settings.xml に以下を追加

    xml
    <settings>
    <localRepository>/path/to/custom-repo</localRepository>
</settings>

  2. install:install-file コマンドで -DlocalRepositoryPath を指定

    sh
    mvn install:install-file \
    -Dfile=/path/to/your-library.jar \
    -DgroupId=com.example \
    -DartifactId=your-library \
    -Dversion=1.0.0 \
    -Dpackaging=jar \
    -DlocalRepositoryPath=/path/to/custom-repo

まとめ

方法メリットデメリット用途
1. ローカルリポジトリに登録 (推奨)Maven の依存管理が適用されるインストールが必要長期的に利用するJAR
2. system スコープで直接指定設定が簡単Mavenの推奨から外れる一時的なJAR利用
3. カスタムローカルリポジトリ~/.m2/repository を汚さずに管理settings.xml の変更が必要特定プロジェクトだけで使う場合

基本的には、方法1（install:install-file でローカルリポジトリに登録） するのが一番良い選択肢です。

方法1: シェルスクリプトで mvn install:install-file をループ実行

JAR ファイルが複数ある場合、スクリプトを使って一括で登録できます。

手順

  1. JAR ファイルを lib ディレクトリに格納する。

    swift
    /path/to/libs/
├── lib1.jar
├── lib2.jar
├── lib3.jar

  2. 以下のスクリプトを実行

    • Linux/macOS (Bashスクリプト)

      sh
      for jar in /path/to/libs/*.jar; do
    artifactId=$(basename "$jar" .jar)
    mvn install:install-file \
        -Dfile="$jar" \
        -DgroupId=com.example.libs \
        -DartifactId="$artifactId" \
        -Dversion=1.0.0 \
        -Dpackaging=jar
done

    • Windows (PowerShell)

      powershell
      Get-ChildItem "C:\path\to\libs\*.jar" | ForEach-Object {
    $artifactId = [System.IO.Path]::GetFileNameWithoutExtension($_.Name)
    mvn install:install-file `
        -Dfile=$_.FullName `
        -DgroupId=com.example.libs `
        -DartifactId=$artifactId `
        -Dversion=1.0.0 `
        -Dpackaging=jar
}

  3. pom.xml に依存関係を追加

    xml
    <dependency>
    <groupId>com.example.libs</groupId>
    <artifactId>lib1</artifactId>
    <version>1.0.0</version>
</dependency>
<dependency>
    <groupId>com.example.libs</groupId>
    <artifactId>lib2</artifactId>
    <version>1.0.0</version>
</dependency>

方法2: Maven の deploy ゴールを利用

複数の JAR をリモートまたはローカルリポジトリに一括でデプロイする方法。

  1. pom.xml を作成 lib-deploy-pom.xml (新規ファイル)

    xml
    <project xmlns="http://maven.apache.org/POM/4.0.0"
         xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance"
         xsi:schemaLocation="http://maven.apache.org/POM/4.0.0 http://maven.apache.org/xsd/maven-4.0.0.xsd">
    <modelVersion>4.0.0</modelVersion>
    <groupId>com.example.libs</groupId>
    <artifactId>lib-deployer</artifactId>
    <version>1.0.0</version>
    <packaging>pom</packaging>
    <build>
        <plugins>
            <plugin>
                <groupId>org.apache.maven.plugins</groupId>
                <artifactId>maven-install-plugin</artifactId>
                <executions>
                    <execution>
                        <phase>validate</phase>
                        <goals>
                            <goal>install-file</goal>
                        </goals>
                        <configuration>
                            <file>${project.basedir}/libs/lib1.jar</file>
                            <groupId>com.example.libs</groupId>
                            <artifactId>lib1</artifactId>
                            <version>1.0.0</version>
                            <packaging>jar</packaging>
                        </configuration>
                    </execution>
                    <execution>
                        <phase>validate</phase>
                        <goals>
                            <goal>install-file</goal>
                        </goals>
                        <configuration>
                            <file>${project.basedir}/libs/lib2.jar</file>
                            <groupId>com.example.libs</groupId>
                            <artifactId>lib2</artifactId>
                            <version>1.0.0</version>
                            <packaging>jar</packaging>
                        </configuration>
                    </execution>
                </executions>
            </plugin>
        </plugins>
    </build>
</project>

  2. Maven コマンドを実行

    sh
    mvn -f lib-deploy-pom.xml install

ChatGPT:

MavenでJARファイルを作成する際に、JARファイル内にソースコードを含める方法はいくつかあります。

方法1: maven-jar-plugin を使用する（単一JARにソースを含める）

通常の JAR に .java ファイルを含めたい場合、maven-jar-plugin を設定して includeSources を有効にします。

設定手順

  1. pom.xml に以下の設定を追加

    xml
    <build>
    <plugins>
        <plugin>
            <groupId>org.apache.maven.plugins</groupId>
            <artifactId>maven-jar-plugin</artifactId>
            <version>3.2.0</version>
            <executions>
                <execution>
                    <id>include-sources</id>
                    <goals>
                        <goal>jar</goal>
                    </goals>
                    <configuration>
                        <includes>
                            <include>**/*.java</include>
                        </includes>
                    </configuration>
                </execution>
            </executions>
        </plugin>
    </plugins>
</build>

  2. Mavenビルドを実行

    sh
    mvn clean package

    これにより、ソースコード（.java）がJAR内に含まれた状態でビルドされます。

方法2: maven-source-plugin を使用する（ソースJARを別ファイルとして作成）

一般的に、通常のJARとソースコード（-sources.jar）を別々に作成することが多いため、maven-source-plugin を使用する方法も推奨されます。

設定手順

  1. pom.xml に以下の設定を追加

    xml
    <build>
    <plugins>
        <plugin>
            <groupId>org.apache.maven.plugins</groupId>
            <artifactId>maven-source-plugin</artifactId>
            <version>3.2.1</version>
            <executions>
                <execution>
                    <id>attach-sources</id>
                    <goals>
                        <goal>jar</goal>
                    </goals>
                </execution>
            </executions>
        </plugin>
    </plugins>
</build>

  2. Mavenビルドを実行

    sh
    mvn clean package

    この設定を有効にすると、通常の JAR (target/myapp-1.0.jar) に加えて、ソースコード入りの JAR (target/myapp-1.0-sources.jar) も作成されます。

方法3: maven-assembly-plugin を使用する（ソース付きのカスタムJARを作成）

特定のJAR形式でソースコードを含めたものを作りたい場合、maven-assembly-plugin を使用できます。

設定手順

  1. pom.xml に以下の設定を追加

    xml
    <build>
    <plugins>
        <plugin>
            <groupId>org.apache.maven.plugins</groupId>
            <artifactId>maven-assembly-plugin</artifactId>
            <version>3.4.2</version>
            <executions>
                <execution>
                    <id>source-jar</id>
                    <phase>package</phase>
                    <goals>
                        <goal>single</goal>
                    </goals>
                    <configuration>
                        <descriptors>
                            <descriptor>src/assembly/source-jar.xml</descriptor>
                        </descriptors>
                    </configuration>
                </execution>
            </executions>
        </plugin>
    </plugins>
</build>

  2. src/assembly/source-jar.xml を作成

    xml
    <assembly xmlns="http://maven.apache.org/ASSEMBLY/2.2.0"
          xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance"
          xsi:schemaLocation="http://maven.apache.org/ASSEMBLY/2.2.0 http://maven.apache.org/xsd/assembly-2.2.0.xsd">
    <id>source-jar</id>
    <formats>
        <format>jar</format>
    </formats>
    <includeBaseDirectory>false</includeBaseDirectory>
    <fileSets>
        <fileSet>
            <directory>src/main/java</directory>
            <outputDirectory>/</outputDirectory>
            <includes>
                <include>**/*.java</include>
            </includes>
        </fileSet>
    </fileSets>
</assembly>

  3. Mavenビルドを実行

    sh
    mvn clean package assembly:single

    target/myapp-1.0-source.jar という名前で、ソースコードが含まれたJARが作成されます。

まとめ

方法出力結果メリットデメリット
方法1: maven-jar-pluginmyapp-1.0.jar（本体JARにソース含む）JARにソースが直接含まれる通常のJARとしては不要な場合が多い
方法2: maven-source-pluginmyapp-1.0.jar myapp-1.0-sources.jar一般的なMavenの標準形式2つのJARが作成される
